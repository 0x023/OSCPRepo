Check for the services that "Log On As" SYSTEM for highest privs. Alternative is a user who is part of the local Administrators group on the box.

In Windows, information related to services is stored in HKLM\SYSTEM\CurrentControlSet\Services registry key. If we want to see information about our “Vulnerable Service” we should check
HKLM\SYSTEM\ControlSet001\Services\Vulnerable Service key.

Permissions on folders can be done with built-in tool: cacls, icacls

Permissions on services can be done with built-in tool: sc sdshow
#the output is very cryptic, need a reference sheet
sc sdshow <ServiceName>

Permissions on services can be set with built-in tool: sc sdset
sc <server> sdset <ServiceName> <SD in SDLL format>

Permissions on services can be done with non-built-in tool: accesschk.exe and subinacl
accesschk.exe -uwcqv "Authenticated Users" * /accepteula
