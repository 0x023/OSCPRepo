Keys are both in Active Directory and client LSASS memory

We can find:
  DES key
  RC4 key (this is the NTLM hash of the password, no domain salt!)
  AES128 and AES 256 keys (with NT 6)

New "Protected Users" group prevents keys in client LSASS memory
  Of course not on the DC itself

AES Keys use PBKDF2
  These hashes are salted
  4096 iterations of the PBKDF2 algorithm
  Difficult to crack

Of course these hashes are cached in memory on the client side and then used as password equivs, just like NT hashes
This is how you fail with strong cryptography
