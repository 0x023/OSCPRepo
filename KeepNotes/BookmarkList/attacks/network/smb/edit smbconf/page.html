If you can edit smb.conf, you have a potential route for elevated access.

If we tell Samba to listen on an interface (essentially enabling it) and adding an “Example” share with guest access, the configuration looks like this:

[Example]
path = /mnt
read only = no
guest ok = yes
create mask = 0700
directory mask = 0700


Try to set the path to / or other privileged folders


Try adding:
force user = root

This will attempt to force all connections as the user you specify (ie root)
